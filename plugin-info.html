Plugin Documentation

Goals available for this plugin:

Goal Description
otka:help Display help information on okta-maven-plugin.
Call mvn otka:help -Ddetail=true -Dgoal=<goal-name> to display parameter details.
otka:jhipster Creates a new Okta OIDC Web Application for use with your JHipster application and writes an .okta.env file with it's configuration.

NOTE: You must have an existing Okta account to use this Mojo, use the goals: okta:register or okta:login.

To create other types of applications on the command line see the Okta CLI.

otka:login Authenticates the Okta CLI and Okta Maven Plugin with an exiting Okta Organization. To sign up for a new or use okta:register.
otka:register Signs up for a free Okta Developer Account. Prompts for basic user info, accounts will be activated via email.
otka:setup Deprecated. Use okta:register, okta:spring-boot, okta:web-app, and okta:jhipster instead.
otka:spring-boot Creates a new Okta OIDC Web Application for use with your Spring Boot application and writes an src/main/application.properties|yml file with it's configuration.

NOTE: You must have an existing Okta account to use this Mojo, use the goals: okta:register or okta:login.

To create other types of applications on the command line see the Okta CLI.

otka:web-app Creates a new Okta OIDC Web Application for use with your Web Application and writes an .okta.env file with it's configuration.

NOTE: You must have an existing Okta account to use this Mojo, use the goals: okta:register or okta:login.

To create other types of applications on the command line see the Okta CLI.

System Requirements

The following specifies the minimum requirements to run this Maven plugin:

Maven 2.0
JDK 1.8
Memory No minimum requirement.
Disk Space No minimum requirement.

Usage

You should specify the version in your project's plugin configuration:

<project>
  ...
  <build>
    <!-- To define the plugin version in your parent POM -->
    <pluginManagement>
      <plugins>
        <plugin>
          <groupId>com.okta</groupId>
          <artifactId>okta-maven-plugin</artifactId>
          <version>0.3.0</version>
        </plugin>
        ...
      </plugins>
    </pluginManagement>
    <!-- To use the plugin goals in your POM or parent POM -->
    <plugins>
      <plugin>
        <groupId>com.okta</groupId>
        <artifactId>okta-maven-plugin</artifactId>
        <version>0.3.0</version>
      </plugin>
      ...
    </plugins>
  </build>
  ...
</project>

For more information, see "Guide to Configuring Plug-ins"
